HOW ZERO-KNOWLEDGE WORKS

  • Your note/file is AES-256 encrypted in the browser. The server only ever receives unreadable ciphertext.
  • The secret key is embedded in the URL fragment (#). Browsers never include fragments in HTTP requests — the server cannot see it.
  • With Burn-on-Read, the server deletes the note before responding to the first fetch — preventing any second access.
  • The auto-wipe timer runs entirely in the viewer's browser. After the countdown, the decrypted content is wiped from screen.
  • Files are base64-encoded and encrypted as a single blob. The server has no idea whether the blob is text or a file.
