How zero-knowledge works

  • Your note or file is AES-256 encrypted in the browser. The server only receives unreadable ciphertext.
  • The secret key is embedded in the URL fragment (#). Browsers never include the fragment in HTTP requests — the server cannot see it.
  • With burn-on-read enabled, the server deletes the note before responding to the first request.
  • The auto-wipe timer runs in the viewer's browser and wipes the decrypted content from screen when it expires.
